### 1. What is Static File Serving
- Returning files that already exists such as HTML, CSS, JS, images, fonts as a request to a client with no additional logics or database access

||Static File|Dynamic File|
|--|--|--|
|Creation|Already exists as a file|Created by logic when requested|
|Example|HTML, CSS, JS, image, font|API response|
|Role of Server|Response exactly|Execute code and geserate result|
|Speed|Fast|(relatively) Slow|
|Caching|Ease|Strategy needed|
|Server|Nginx, CDN, Express.static|Node.js, Django, NestJS|

### 2. Why Static Serving is Important?
- Decreases burden of a server
- Enhances speed of response
- Enforces sequrity
- Saves money
